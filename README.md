## рҹ—ӮпёҸ `git log --projects` вҖ” Featured Case Studies

<br/>

> [!IMPORTANT]
> ## рҹӣ’ B2B E-Commerce Platform вҖ” Enterprise Grade
>
> **`Next.js 14` В· `TypeScript` В· `TanStack Query v5` В· `Zustand` В· `Shadcn UI` В· `Tailwind CSS` В· `Zod` В· `Framer Motion`**
>
> ### Problem Statement
> Build a scalable B2B storefront that handles 1000s of product SKUs, multi-vendor catalogs, real-time cart sync, and role-based purchasing flows вҖ” without the UI ever feeling slow.
>
> ### Architecture Decisions
>
> **Server-State (TanStack Query):**
> - `useInfiniteQuery` for paginated product catalogs with cursor-based pagination
> - Prefetching product details on hover with `queryClient.prefetchQuery`
> - Cart mutations with full optimistic updates via `onMutate` вҶ’ `onError` rollback
> - Background refetch on window focus to keep inventory counts fresh
> - Custom query keys factory for predictable cache invalidation
>
> **Client-State (Zustand):**
> - Slices architecture: `cartSlice`, `filterSlice`, `uiSlice`, `authSlice`
> - Persisted cart with `zustand/middleware/persist` + localStorage hydration
> - Immer middleware for deeply nested state mutations
>
> **Performance Wins:**
> - `React.memo` + `useMemo` on product grid items вҶ’ **60% fewer re-renders**
> - Route-level code splitting with Next.js dynamic imports
> - Zod schema at API boundary вҖ” zero runtime type errors in production
>
> **Result:** Sub-100ms UI interactions. Zero loading spinners for repeat navigation.

<br/>

> [!TIP]
> ## рҹҗҰ Twitter/X Clone вҖ” Real-time Social Architecture
>
> **`React 18` В· `TypeScript` В· `TanStack Query` В· `Zustand` В· `Framer Motion` В· `Socket.io` В· `Tailwind CSS` В· `Shadcn UI`**
>
> ### Problem Statement
> Replicate Twitter's real-time feel: instant likes, seamless replies, smooth feed loading, and WebSocket-based notifications вҖ” all while maintaining predictable state.
>
> ### Architecture Decisions
>
> **Optimistic UI (The Crown Jewel):**
> - Every mutation (like, repost, follow, reply) is instant via TanStack's `onMutate`
> - Previous cache snapshot saved вҶ’ rolled back with `onError` if server rejects
> - `onSettled` refetch for eventual consistency
>
> **Real-time Layer:**
> - Socket.io for live notifications and online presence indicators
> - Zustand stores the active socket instance globally
> - Message queue pattern for race condition prevention
>
> **Feed Architecture:**
> - `useInfiniteQuery` with Intersection Observer for seamless infinite scroll
> - Stale-while-revalidate: feed feels instant on re-visit
> - Framer Motion `AnimatePresence` for tweet enter/exit animations
>
> **Result:** Twitter-level responsiveness. Zero perceived lag on interactions.
